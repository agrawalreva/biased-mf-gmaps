Predictive Task: Rating Prediction on Google Maps Reviews

The goal of this project is to build a personalized recommender system that predicts a user's future rating for a business in California. Formally, given a user (u) and an item (i), the model estimates the rating (r_{u,i}) they would give in the future.

This is a collaborative filtering task uses this dataset: https://mcauleylab.ucsd.edu/public_datasets/gdrive/googlelocal/.

Each review contains:

  • user_id
  • item_id (business)
  • rating (15 stars)
  • time (timestamp)
  • text (optional review text)

The dataset is 10-core filtered, meaning every user and item has at least 10 interactions.
This makes the dataset suitable for latent factor models.

Significance

Accurate rating prediction powers recommender systems such as Google Maps, Yelp, and Amazon. Personalized predictions:

  • Improve recommendation quality
  • Help users discover relevant businesses
  • Reduce information overload
  • Enable Top-K recommendations
  • Support ranking tasks

In this notebook, we evaluate both simple mean-based baselines and advanced matrix factorization models. We also evaluate the ranking performance of these models using Top-K metrics and perform an ablation study to understand long-tail effects.

def parse_json_gz(path, max_lines=None):
    with gzip.open(path, 'rt', encoding='utf8') as g:
        for i, line in enumerate(g):
            if max_lines and i >= max_lines:
                break
            line = line.strip()
            if not line:
                continue
            try:
                yield json.loads(line)
            except Exception as e:
                continue

def load_reviews_to_df(path, nrows=None):
    rows = []
    for rec in tqdm(parse_json_gz(path, max_lines=nrows), desc="Loading reviews"):
        row = {}
        row['user_id'] = rec.get('user_id')
        rid = rec.get('gmap_id') or rec.get('gmapId') or rec.get('gmapid') or rec.get('business_id') or rec.get('place_id')
        row['item_id'] = rid
        row['rating'] = rec.get('rating')
        row['time'] = rec.get('time')
        row['text'] = rec.get('text') if 'text' in rec else (rec.get('review_text') or None)
        rows.append(row)
    df = pd.DataFrame(rows)

    if 'time' in df.columns:
        df['time'] = pd.to_numeric(df['time'], errors='coerce')
        df['time'] = pd.to_datetime(df['time'], unit='ms', errors='coerce')
        
    return df

def preprocess_ratings_df(ratings_df):
    df = ratings_df.copy()
    
    df = df.dropna(subset=['user_id', 'item_id', 'rating'])
    
    df['rating'] = df['rating'].astype(float) 

    unique_users = sorted(df["user_id"].unique())
    unique_items = sorted(df["item_id"].unique())

    user2idx = {u: idx for idx, u in enumerate(unique_users)}
    item2idx = {i: idx for idx, i in enumerate(unique_items)}

    df["user_idx"] = df["user_id"].map(user2idx).astype(int)
    df["item_idx"] = df["item_id"].map(item2idx).astype(int)

    return df, user2idx, item2idx

def time_based_split(df_in, test_frac=0.2):
    df = df_in.sort_values("time").copy()
    train_rows = []
    test_rows = []
    
    for u, user_df in tqdm(df.groupby("user_idx"), desc="Splitting by User"):
        n = len(user_df)
        k = max(1, int(np.ceil(test_frac * n)))
        
        test_rows.append(user_df.tail(k))
        train_rows.append(user_df.head(n - k))

    train_df = pd.concat(train_rows).reset_index(drop=True)
    test_df = pd.concat(test_rows).reset_index(drop=True)
    
    return train_df, test_df

def build_sparse_matrix(df, n_users, n_items):
    rows = df['user_idx'].to_numpy()
    cols = df['item_idx'].to_numpy()
    data = df['rating'].to_numpy(dtype=float)
    mat = sps.csr_matrix((data, (rows, cols)), shape=(n_users, n_items))
    return mat

def rmse_from_mse_or_arrays(y_true, y_pred):
    return np.sqrt(np.mean((y_true - y_pred)**2))

def eval_regression_baselines(train_df, test_df):
    
    y_true = test_df["rating"].values
    global_mean = train_df["rating"].mean()
    
    y_pred_global = np.full_like(y_true, global_mean, dtype=float)
    
    user_means = train_df.groupby("user_idx")["rating"].mean()
    y_pred_user = test_df["user_idx"].map(user_means).fillna(global_mean).values
    
    item_means = train_df.groupby("item_idx")["rating"].mean()
    y_pred_item = test_df["item_idx"].map(item_means).fillna(global_mean).values
    
    results = {}
    for name, y_pred in {'Global Mean': y_pred_global, 'User Mean': y_pred_user, 'Item Mean': y_pred_item}.items():
        results[name] = {
            "RMSE": rmse_from_mse_or_arrays(y_true, y_pred),
            "MAE": mean_absolute_error(y_true, y_pred)
        }
        
    return pd.DataFrame(results).T.sort_values(by="RMSE")

def train_biased_mf_sgd(train_df, n_users, n_items, k=32, lr=0.01, reg=0.05, n_epochs=20, seed=RANDOM_SEED):
    np.random.seed(seed)
    
    global_mean = train_df['rating'].mean()
    U = 0.01 * np.random.randn(n_users, k)
    V = 0.01 * np.random.randn(n_items, k)
    
    b_u = np.zeros(n_users)
    b_i = np.zeros(n_items)
    
    n_epochs = 20 

    rows = train_df[['user_idx', 'item_idx', 'rating']].to_numpy()
    
    print(f"Starting Biased MF-SGD Training (k={k}, lr={lr}, reg={reg}, epochs={n_epochs})...")
    
    for epoch in range(n_epochs):
        np.random.shuffle(rows)
        loss_accum = 0.0
        
        for ui, ii, r in rows:
            ui, ii = int(ui), int(ii)
            
            pred = global_mean + b_u[ui] + b_i[ii] + U[ui].dot(V[ii])
            e = r - pred

            b_u[ui] += lr * (e - reg * b_u[ui])
            b_i[ii] += lr * (e - reg * b_i[ii])
            
            U[ui] += lr * (e * V[ii] - reg * U[ui])
            V[ii] += lr * (e * U[ui] - reg * V[ii])
            
            loss_accum += e*e
            
        print(f"Epoch {epoch+1}/{n_epochs} Training MSE: {loss_accum/len(rows):.4f}")
        
    return global_mean, U, V, b_u, b_i

def svd_baseline(train_mat, k=40):
    
    coo = train_mat.tocoo()
    global_mean = coo.data.mean()
    
    R_centered = train_mat.copy().astype(float)
    R_centered.data = R_centered.data - global_mean
    
    u, s, vt = svds(R_centered, k=k)
    
    s_sorted_idx = np.argsort(s)[::-1]
    s_sorted = s[s_sorted_idx]
    u_sorted = u[:, s_sorted_idx]
    vt_sorted = vt[s_sorted_idx, :]
    
    sqrt_s = np.sqrt(np.abs(s_sorted))
    U_factors = u_sorted * sqrt_s[np.newaxis, :]
    V_factors = (vt_sorted.T) * sqrt_s[np.newaxis, :]
    
    return global_mean, U_factors, V_factors

def predict_ratings_from_factors(U, V, df, mu=0.0, b_u=None, b_i=None):

    preds = []
    
    if b_u is None: b_u = np.zeros(U.shape[0])
    if b_i is None: b_i = np.zeros(V.shape[0])
        
    for ui, ii in df[['user_idx','item_idx']].to_numpy():
        ui, ii = int(ui), int(ii)
        
        if ui < U.shape[0] and ii < V.shape[0]:
            prediction = mu + b_u[ui] + b_i[ii] + U[ui].dot(V[ii])
            preds.append(prediction)
        else:
            preds.append(mu) 
            
    return np.array(preds)

Popularity Baseline for Top-K Recommendation

In addition to regression evaluation, we compare Top-K recommendation quality using a Popularity baseline.
This non-personalized model ranks items only by their frequency in the training dataset.

For every user, it recommends the same Top-K most popular items:

$ \text{Rec}_u = \text{TopK}(\text{most frequent items}) $

Popularity is a strong classical baseline because it reflects the minimal performance that any personalized model should exceed.

In sparse datasets like Google Maps reviews, it is expected that:

  • Popularity gives very low precision and NDCG
  • Users often rate niche businesses
  • The MF model should still outperform popularity

We compare Popularity vs. MF-SGD using Precision@K and NDCG@K.

def precision_at_k(r_true_binary, r_scores, k):
    idx = np.argsort(-r_scores)[:k]
    return r_true_binary[idx].sum() / k

def recall_at_k(r_true_binary, r_scores, k):
    idx = np.argsort(-r_scores)[:k]
    total_relevant = max(1, r_true_binary.sum()) 
    return r_true_binary[idx].sum() / total_relevant

def dcg_at_k(r_true_graded, r_scores, k):
    idx = np.argsort(-r_scores)[:k]
    gains = r_true_graded[idx]
    discounts = np.log2(np.arange(2, k + 2))
    return (gains / discounts).sum()

def idcg_at_k(r_true_graded, k):
    gains_sorted = np.sort(r_true_graded)[::-1]
    discounts = np.log2(np.arange(2, k + 2))
    return (gains_sorted[:k] / discounts).sum()

def ndcg_at_k(r_true_graded, r_scores, k):
    dcg = dcg_at_k(r_true_graded, r_scores, k)
    idcg = idcg_at_k(r_true_graded, k)
    return dcg / idcg if idcg > 0 else 0.0

def evaluate_topk_biased(model_type, U, V, test_df, train_df, k_list=[5, 10], mu=0.0, b_u=None, b_i=None):
    n_items = V.shape[0]
    train_pos = train_df.groupby('user_idx')['item_idx'].apply(set).to_dict()
    test_data = test_df.groupby('user_idx')
    users = sorted(test_data.groups.keys())
    out = {k: {'precision': [], 'recall': [], 'ndcg': []} for k in k_list}
    
    if model_type == 'Popularity':
        scores_pop = train_df.groupby('item_idx').size().reindex(np.arange(n_items), fill_value=0).values.astype(float)
        
    for u in tqdm(users, desc=f"Eval {model_type} Top-K"):
        
        if model_type == 'Popularity':
            scores = scores_pop
        else:
            ui = u
            prediction_base = mu + b_u[ui] + b_i 
            scores = prediction_base + V.dot(U[ui])

        train_items = train_pos.get(u, set())
        scores[list(train_items)] = -np.inf

def head_tail_split(train_df, q=0.8):
    per_item_counts = train_df['item_idx'].value_counts()
    threshold = per_item_counts.quantile(q)
    
    head_items = set(per_item_counts[per_item_counts >= threshold].index)
    tail_items = set(per_item_counts[per_item_counts < threshold].index)
    return head_items, tail_items

Ablation Study Discussion: Head vs Tail

Items were split into:

  • Head items (top 20% most-reviewed)
  • Tail items (bottom 80% least-reviewed)

Results show:

  • RMSE on Head items is lower
  • RMSE on Tail items is higher

This is the long-tail effect:

  1. Head items benefit from more interactions, improving latent factor learning.
  2. Tail items have fewer signals, making it harder for MF to learn reliable embeddings.
  3. This performance gap is expected and aligns with modern recommendation literature.

The ablation confirms data sparsity remains a core challenge for collaborative filtering models.

Related Work

Recommender systems have evolved from simple heuristics to modern latent factor and neural models.

Collaborative Filtering

Early systems used nearest neighbor methods (user-based or item-based).
These approaches are easy to interpret but struggle with sparsity and scalability (Sarwar et al., 2001).

Matrix Factorization

The Netflix Prize popularized biased matrix factorization (Koren et al., 2009), using the prediction:

$ \hat{r}_{u,i} = \mu + b_u + b_i + \mathbf{u}_u^\top \mathbf{v}_i $

This method captures user preferences and item characteristics, and remains one of the strongest classical CF approaches.

Truncated SVD

SVD factorizes the mean-centered rating matrix.
While efficient, it does not include user/item biases and typically underperforms MF in rating prediction.

Top-K Ranking Metrics

Metrics such as Precision@K and NDCG@K (He et al., 2017) evaluate top-N recommendation quality.
These align with real-world recommender tasks and are used in this project.

Long-Tail Behavior

Real datasets exhibit strong popularity imbalance.
Items in the tail have fewer interactions, making them harder to model.
This motivates our ablation study comparing head vs tail performance.

Hybrid Models

Recent systems incorporate textual review features (TF-IDF, CNNs, transformers) to represent item semantics (McAuley et al., 2015).
This is a natural extension for improving predictions on sparse or long-tail items.
